# Documentation Guide

The documents in this directory describe the architecture and conventions of the SRYSO Repertoire Project.

Each document has a distinct purpose.

| Document | Purpose |
|---|---|
| ARCHITECTURE.md | Why is the repository designed this way? |
| DIRECTORY-CONVENTIONS.md | Where does everything live? |
| REPOSITORY-CONVENTIONS.md | How do we name and catalog things? |
| MIGRATION-WORKFLOW.md | How do migrations proceed? |
| PROJECT-DNA.md | What principles guide the project? |
| SPREADSHEET-FIELDS.md | What does each spreadsheet field represent? |

The goal of this documentation is not to duplicate the implementation.

Rather, it explains the design decisions behind the repository so that future maintainers —- including future versions of the authors -— can understand the reasoning before modifying the system.
